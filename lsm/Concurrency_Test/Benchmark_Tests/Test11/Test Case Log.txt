Multiple Write MRead Interactions

Expected: All the write operations will run then all the mread operations will run. 
Actual: Since the write from T0 locks the file preventing the mreads, all the mreads on the file is blocked. Since the same lock also block any other erases to run on the record level, T0 is the only one allowed to run. As we get the transactions from other files we just put them in the delaylist for each transaction. Then onces T0 commits, each transaction will run all their operations before moving on to the next.
